Goals of the portal

With the need to improve project performance, it has become apparent that project teams need a familiar, consistent mechanism to access project documentation while in the field. To address this, the following requirements were in mind:
  • Must work in an offshore environment. This can mean many things. In this case it means without ICT support for web servers (IIS, Apache) which require some skill to opertate.
  • Must work from a thumb drive. This means static HTML code, no server side scripting, no PHP, no RDBMS, No Apache, No IIS (thank god)
  • User needs to be able to find documents (procedures, forms etc) with ease using built in 'find in page' tool in all browsers.
  • Links need to be ultra reliable (auto generated is good for this).
  • Needs to be HTML5 compliant so it can be hosted on the cloud in the future.
  • Needs to work (and tested) with both IE and Chrome.
  • Needs to be fast for users
  • Needs to be very easy for project leads, and indeed any tema mamber to update (as easy as a double click)
  • Most of all, it needs to be useful!

How to edit this portal

  • Having project documents in a folder system is easy for the authors, but can be remarkably difficult for users to find things. The project portal permits the document authors to use a folder system to compile all project documentation into a single location.
  • To build the web portal, we use an automated scan-and-publish script, written in python. If you have never used python youtube is your friend.
  • To run a python script, you need to have python 3.4 or higher installed on your pc. download python here: python download
  • To build your project documentation, simply double click on the 'buildProjectDocumentatiton.py' script in the root folder of your portal.
  • running the build process will automatically scan all subfolders, compile a list of any and all documents in those folders, create a series of links and burn them into the index.html page in the root of the project
  • It will only take a few seconds to rebuild your portal. This means you can do it every time you add a new document.
  • This can be used to add daily operatins reports, toolbox talks, weather reports, anything.
  • If you wish to publish the portal across the entire vessel, an opensource web server such as nginx is recommended. This is very esy to use and only takes moments to install.
